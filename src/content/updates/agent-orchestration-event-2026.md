We've been kicking around the idea of doing an **Agent Orchestration Night** and wanted to see if there's interest.

The concept: an evening where people demo how they're wiring up multiple AI agents to work together. Not a talk, not a panel. Just people showing their setups and us all figuring out what actually works.

## What we mean by agent orchestration

If you've used a coding agent that can spawn sub-agents, or built a pipeline where one model does research and another writes code based on it, or set up MCP servers so your agents can use external tools — that's the kind of thing we're talking about. Multiple agents coordinating on a task instead of one model doing everything.

Some stuff we'd love to see demos of:

- Multi-agent pipelines (agents handing off work to each other)
- MCP (Model Context Protocol) setups and custom tool servers
- Orchestration frameworks you've tried — what worked, what didn't
- Failures are welcome. Honestly, those are usually the best ones.

## Why now?

We've been running our own agent orchestration setup with [SAM](https://github.com/raphaeltm/simple-agent-manager) to manage this website, plan events, and coordinate across repos. A bunch of regulars at the meetup are doing similar things at work. Felt like a good excuse to get together and compare notes over a beer.

## Want in?

No date or venue yet. We want to gauge interest first. If you'd want to come — or even better, if you've got something to demo — tell us at a Thursday meetup or ping us on [Meetup](https://www.meetup.com/biere-code-beer-paris/).

More details once we figure out the logistics.
